3. 주제별 보기
# 한 주제보기 알고리즘 정리

## 1. 목표

한 주제(예: 사랑)를 선택했을 때  
처음에는 핵심 구절 몇 개만 보여주고,  
사용자의 실제 행동을 기반으로 추천 순서를 조금씩 조정한다.

핵심 원칙은 다음과 같다.

- 처음에는 적게 보여준다
- 대표 구절을 먼저 보여준다
- 더 보기 / 모두 보기로 확장한다
- 점수는 단순 클릭이 아니라 읽기 깊이에 따라 반영한다

5. N개 선택 알고리즘
5.1 목표

입력:

주제(category)
보여줄 개수(N)
사용자 정보(userId)
이미 본 구절 목록(seenVerseIds)

출력:

추천 구절 N개
5.2 선택 원칙
대표 구절 1개는 포함한다
나머지는 랜덤이지만 너무 치우치지 않게 고른다
최근 본 구절은 가능하면 제외한다
같은 책만 몰리지 않게 분산한다
5.3 최종 추천 점수

각 구절의 추천 점수는 다음처럼 계산한다.

finalWeight = baseWeight + score + recentScore

예시:

baseWeight: 초기 대표성
score: 누적 사용자 행동
recentScore: 최근 며칠간 반응
5.4 N개 선택 절차
Step 1. 주제 필터
const pool = verses.filter(v => v.mainCategory === category)
Step 2. 최근 본 구절 제외
const filteredPool = pool.filter(v => !seenVerseIds.includes(v.verseId))

최근 본 구절을 제외했는데 너무 적으면 전체 pool을 다시 사용한다.

Step 3. 대표 구절 1개 선택

대표 구절은 다음 우선순위로 고른다.

isAnchor === true
finalWeight가 높은 구절
최근 너무 자주 노출되지 않은 구절
const anchors = filteredPool.filter(v => v.isAnchor)
const mainPick = weightedRandom(anchors.length ? anchors : filteredPool)
Step 4. 나머지 후보 구성

대표 구절과 같은 책의 구절이 너무 많지 않도록 일부 제한한다.

let candidates = filteredPool.filter(v => v.verseId !== mainPick.verseId)
Step 5. 나머지 N-1개 선택
const result = [mainPick]

while (result.length < N && candidates.length > 0) {
  const pick = weightedRandom(candidates)

  result.push(pick)

  candidates = candidates.filter(v => v.verseId !== pick.verseId)
}
Step 6. 다양성 보정

선택 결과가 같은 책에 몰렸으면 일부를 다른 책 구절로 교체한다.

5.5 weightedRandom 함수
function weightedRandom(items) {
  const total = items.reduce((sum, item) => sum + getFinalWeight(item), 0)
  let rand = Math.random() * total

  for (const item of items) {
    rand -= getFinalWeight(item)
    if (rand <= 0) return item
  }

  return items[items.length - 1]
}
function getFinalWeight(item) {
  return item.baseWeight + item.score + item.recentScore
}
5.6 전체 예시 함수
function getTopicVerses({ category, N, seenVerseIds = [] }) {
  const pool = verses.filter(v => v.mainCategory === category)

  let filteredPool = pool.filter(v => !seenVerseIds.includes(v.verseId))
  if (filteredPool.length < N) filteredPool = pool

  const anchors = filteredPool.filter(v => v.isAnchor)
  const mainPick = weightedRandom(anchors.length ? anchors : filteredPool)

  let candidates = filteredPool.filter(v => v.verseId !== mainPick.verseId)

  const result = [mainPick]

  while (result.length < N && candidates.length > 0) {
    const pick = weightedRandom(candidates)
    result.push(pick)
    candidates = candidates.filter(v => v.verseId !== pick.verseId)
  }

  return result
}
6. 더 보기 알고리즘
6.1 목적

초기 3개를 본 뒤
추가로 3개 또는 5개를 더 보여준다.

6.2 원칙
이미 본 구절 제외
너무 비슷한 구절 반복 방지
가능하면 다른 책, 다른 흐름도 섞기
6.3 예시
function getMoreTopicVerses({ category, N, alreadyShownIds = [] }) {
  return getTopicVerses({
    category,
    N,
    seenVerseIds: alreadyShownIds
  })
}

7.2 정렬 기준

모두 보기에서는 다음 기준으로 정렬한다.

핵심 구절 먼저
finalWeight 높은 순
성경 순서 기준 보조 정렬
7.3 예시
function getAllTopicVerses(category) {
  return verses
    .filter(v => v.mainCategory === category)
    .sort((a, b) => {
      if (a.isAnchor !== b.isAnchor) return a.isAnchor ? -1 : 1
      const diff = getFinalWeight(b) - getFinalWeight(a)
      if (diff !== 0) return diff
      return getBibleOrder(a) - getBibleOrder(b)
    })
}
8. 행동으로 score update 하는 알고리즘
8.1 핵심 원칙

점수는 단순 클릭이 아니라
얼마나 깊게 읽었는가를 반영해야 한다.

즉,

구절 눌렀다 = 아주 약한 신호
본문 읽었다 = 의미 있는 신호
나눔 봤다 = 더 깊은 신호
묵상 작성했다 = 가장 강한 신호
8.2 행동별 점수 기준

예시 기준:

구절 클릭        +0.5
본문 읽기        +1
나눔 보기        +2
묵상 작성        +3
다시 보기        +1.5
즐겨찾기         +2

초기에는 너무 복잡하게 가지 말고
아래 3개만 써도 충분하다.

본문 읽기        +1
나눔 보기        +2
묵상 작성        +3
8.3 score 업데이트 예시
function updateVerseScore(verseId, actionType) {
  const verse = findVerseById(verseId)
  if (!verse) return

  switch (actionType) {
    case "read":
      verse.score += 1
      verse.recentScore += 1
      break
    case "view_reflection":
      verse.score += 2
      verse.recentScore += 2
      break
    case "write_reflection":
      verse.score += 3
      verse.recentScore += 3
      break
  }

  saveVerse(verse)
}
8.4 행동 이벤트 예시
본문 열었을 때
updateVerseScore("JHN_3_16", "read")
나눔 보기 눌렀을 때
updateVerseScore("JHN_3_16", "view_reflection")
묵상 저장했을 때
updateVerseScore("JHN_3_16", "write_reflection")
8.5 중복 클릭 방지

같은 사용자가 짧은 시간 안에 반복 클릭하는 경우
점수를 계속 올리면 왜곡된다.

예시:

같은 구절
같은 행동
10분 이내
같은 사용자

이면 점수 반영을 제한한다.

function canUpdateScore(userId, verseId, actionType) {
  const recentLog = findRecentActionLog(userId, verseId, actionType, 10) // 10분
  return !recentLog
}
function handleAction(userId, verseId, actionType) {
  if (!canUpdateScore(userId, verseId, actionType)) return

  updateVerseScore(verseId, actionType)
  saveActionLog(userId, verseId, actionType)
}
8.6 시간 감쇠 알고리즘

오래된 인기 구절만 계속 올라오지 않게
최근 점수는 시간이 지나면 조금씩 줄인다.

예시:

function decayRecentScore() {
  for (const verse of verses) {
    verse.recentScore = verse.recentScore * 0.9
    saveVerse(verse)
  }
}

이 작업은 하루 1번 또는 주기적으로 실행한다.

9. 한 주제보기 전체 알고리즘
9.1 초기 진입

사용자가 사랑 주제를 선택한다.

사랑
→ 초기 3개 구절 추천

알고리즘:

mainCategory가 사랑인 구절만 필터
finalWeight 기준으로 대표 구절 1개 선택
나머지 2개는 weighted random
최근 본 구절은 가급적 제외
9.2 구절 클릭

사용자가 구절을 선택한다.

구절 클릭
→ 본문 표시
→ read 점수 반영
9.3 나눔 보기

사용자가 나눔 보기 버튼을 누른다.

나눔 보기
→ 관련 나눔 메시지 표시
→ view_reflection 점수 반영
9.4 묵상 작성

사용자가 묵상을 작성한다.

묵상 저장
→ write_reflection 점수 반영
9.5 더 보기

사용자가 더 보기 버튼을 누른다.

더 보기
→ 이미 본 구절 제외
→ 추가 N개 추천
9.6 모두 보기

사용자가 모두 보기를 누른다.

모두 보기
→ 해당 주제 전체 구절 목록
→ 핵심 구절 먼저
→ finalWeight 높은 순

12. 핵심 정리
핵심 문장 1

처음에는 우리가 정한 핵심 구절이 기준이 된다.

핵심 문장 2

점수는 클릭 수가 아니라 읽기 깊이로 쌓는다.

핵심 문장 3

추천은 랜덤이 아니라 대표성과 경험이 섞인 선택이어야 한다.

핵심 문장 4

한 주제보기는 정보 목록이 아니라 묵상을 시작하게 하는 구조여야 한다.
