## 목적

앱의 화면 흐름과 주요 인터랙션을 빠르게 검토하고 제품 구조를 시각적으로 정리하기 위한 문서.

## 상태

- 초안
- 마지막 검토일: 2026-04-03

## 열린 질문

- 현재 와이어프레임 흐름이 실제 사용자 여정과 맞는가?
- 첫 화면에서 전달해야 할 핵심 행동은 무엇인가?
- 공유, 회고, PDF 기능 중 어느 화면을 먼저 구체화할 것인가?
# 모줄성 전체 화면 흐름 와이어프레임

## 🧭 전체 흐름

```
[홈]
 ↓
[시작하기]
 ↓
[읽기 화면]
 ↓
[색칠]
 ↓
[한 줄 입력]
 ↓
 ├── 나누기 (이미지 생성 → 저장/복사 → SNS)
 ├── 돌아보기
 ├── 생각 나누기
 └── 함께 써보기 (PDF → 종이 활동)
```

---

## 🏠 홈

```
[로고]

오늘,
무엇이 보이셨나요?

[시작하기]
[말씀 펼쳐보기]
[주제로 보기]

[돌아보기]
[생각 나누기]
[함께 써보기]
```

---

## 📖 읽기 화면

```
[창세기 1장]  ← →

혼돈 속에서
질서가 시작된다

-------------------

본문 (문단 클릭 가능)

-------------------

[색 선택]
● ● ● ● ● ● ● ●

-------------------

[나누기]
```

---

## ✍️ 한 줄 입력

```
이 구절에서
무엇이 보였나요?

[______________]

[저장]
```

---

## 🔗 나누기 (공유 준비)

```
[이미지 미리보기]

[한 줄 입력]

[이미지 저장]
[문구 복사]
[공유하기]
```

---

## 🌳 생각 나누기

```
창세기 1장

혼돈 → 빛 → 질서

-------------------

[1:3–5 빛]

"빛은 이미 있던 것이 아니라..."
"나는 지금 무엇을 보고 있는가"

-------------------

[나도 한 줄 남기기]
```

---

## 🔥 핵심 UX 흐름 요약

```
읽기
→ 색칠
→ 한 줄

↓

나누기 / 돌아보기 / 생각 나누기 / 함께 써보기
```

---

## 🔥 핵심 원칙

- 설명하지 않는다 → 경험하게 만든다
- 클릭 2번으로 끝낸다
- 공유는 준비만 한다 (업로드는 사용자)
- 반복 구조를 만든다

👉 “한 구절에서 시작해 전체 흐름으로 확장된다”
