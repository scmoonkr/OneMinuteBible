# 성경 읽기(색칠하기) 지침

## 목적

모줄성 MVP의 성경 읽기와 색칠하기 동작을 구현 기준으로 정리하는 문서.

## 상태

- 확정
- 작성일: 2026-04-03

## 1. 기본 원칙

- 읽기는 `bibleParagraphs`의 블록 구조를 기준으로 진행한다.
- 사용자는 절을 읽고 색을 선택한 뒤 절을 눌러 색칠한다.
- 색칠은 원본 성경 데이터 수정이 아니라 사용자 선택 레이어로 관리한다.

## 2. 데이터 기준

- 원본 category는 `bibleParagraphs.paragraphs[].verses[].category`를 사용한다.
- 사용자 선택 category는 별도 목록으로 관리한다.
- 현재 MVP에서는 사용자 선택값을 `paintedCategories`에 저장한다.
- 원본 `bibleParagraphs`는 수정하지 않는다.
- 식별자는 `bookNo`, `chapterNo`, `paragraphNo`, `verseNo`로 통일한다.
- `paintedCategories` 저장 기준은 `userId`, `verseRange`, `verseIDs`, `updatedAt`이다.
- `verseRange`는 표시용 문자열이고 형식은 `창1:1,3-4,5`를 사용한다.
- `verseIDs`는 처리용 배열이며 예시는 `[1,2,4,5]`이다.

## 3. 화면 표시 규칙

### 기본 화면

- 기본 상태에서는 카테고리 이름을 숨긴다.
- 기본 상태에서는 색칠하지 않은 절의 배경색을 보여주지 않는다.
- 사용자가 색칠한 절만 선택한 색 배경으로 표시한다.

### 카테고리 보기

- `카테고리 보기`를 누르면 원본 `bibleParagraphs`의 category 기준으로 보여준다.
- 이 모드에서는 원래 category 이름과 원래 배경색을 표시한다.
- 사용자가 별도로 덮어쓴 category가 있어도, 보기 모드에서는 원본 category가 보여야 한다.

### 카테고리 숨기기

- `카테고리 숨기기`를 누르면 다시 사용자가 실제로 색칠한 상태 화면으로 돌아간다.
- 색 팔레트 버튼을 눌러 새 category를 선택하는 경우에도 자동으로 사용자 색칠 상태 화면으로 돌아간다.

## 4. say 절 규칙

- `say: true` 절도 일반 절처럼 선택 가능하다.
- `say: true`의 차이는 글자색만 red로 표시하는 것이다.
- `say: true`라고 해서 선택 불가 상태가 되어서는 안 된다.
- `say: true`에도 사용자가 category를 선택해서 색칠할 수 있다.

## 5. 색칠 동작

- 먼저 12 Color Table에서 색(category)을 선택한다.
- 그 다음 절을 누르면 해당 절이 선택한 category로 색칠된다.
- 선택한 색은 사용자 선택값으로 저장한다.
- 현재 장 초기화는 사용자 선택값만 비운다.

## 6. 나눔 규칙

### 기본 상태

- 나눔 list에는 내가 쓴 나눔만 보여준다.

### 나눔 버튼

- `나눔` 버튼은 토글이다.
- 누르면 나눔 list 전체를 보여준다.
- 전체보기에는 다른 사람 나눔 + 내 나눔이 함께 보인다.
- 다시 누르면 기본 상태로 돌아가며, 내부적으로는 `list.find(el => el.내것)`처럼 내것만 보이는 상태가 된다.

### 한 줄 나누기

- `한 줄 나누기`를 하면 내 나눔이 저장된다.
- 내 나눔은 기본 상태에서도 항상 보여야 한다.
- 내 나눔은 전체 나눔 목록을 볼 때도 함께 보여야 한다.
- 나눔 저장 기준은 `bookNo`, `chapterNo`, `paragraphNo`, `verseRange`, `text`, `mine`, `updatedAt`이다.
- `blockId`는 사용하지 않고 `paragraphNo`로 통일한다.

## 7. 선택한 절 표시 규칙

- 나눔 저장 시 선택한 절 범위를 함께 저장한다.
- 표시 형식은 다음과 같다.
- `창1:1`
- `창1:1-2`
- `창1:1,3-4,5,6-7`

- 연속된 절은 범위로 묶고, 끊어진 절은 쉼표로 나눈다.
- 같은 선택 범위에 대해 실제 처리용 값으로 `verseIDs`도 함께 가진다.

## 8. SNS 공유 규칙

- SNS 공유는 내가 색칠한 절 기준으로 만든다.
- 공유 메시지에는 선택한 절 범위와 나눔 내용, 태그를 포함한다.
- 공유 이미지는 사용자가 색칠한 상태를 반영한다.

## 9. 구현 체크포인트

1. 원본 category와 사용자 선택 category가 분리되어 있는가
2. `카테고리 보기`에서 원본 category가 보이는가
3. `say: true` 절이 red 텍스트이면서도 선택 가능한가
4. 기본 나눔 list가 내것만 보여주는가
5. `나눔` 토글 시 전체 목록이 보이는가
6. 선택한 절 범위와 `verseIDs`가 함께 저장되는가
