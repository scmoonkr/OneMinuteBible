# Agent.md – 모줄성 프로젝트 정리

## ✅ 결정사항

### 1. 핵심 개념
- 모줄성 = 색으로 성경을 읽는 방식
- 읽기 → 의미 판단 → 색칠 → 반복 구조
- 성경을 12가지 주제로 분류하여 이해 중심 읽기

### 2. 책 디자인 방향
- 좌: 성경 본문 (줄치기 중심)
- 우: 문단별 짧은 설명 + 여백
- NOTE 줄 구조 제거 → 자유 여백으로 변경
- 절 기준이 아니라 “의미 단위”로 분리
- 같은 절 내 주제 다르면 줄바꿈
- 주제는 블록 아래에 표시 (약하게)

### 3. 워크북 전략
- PDF = 단순 자료 ❌ → “출판물 수준 워크북” ⭕
- 표지 / 여백 / 반복 구조 필수
- 무료 체험용 워크북 제공 (창세기 1장 등)
- 프린트 경험도 책처럼 느껴지게 설계

### 4. 출판 전략
- 초기: 창세기, 마태복음 중심 소량 출간
- 나머지: 워크북 PDF로 제공
- 점진적 확장

### 5. 웹 서비스 방향
- 책 중심, 웹은 보조
- 기능:
  - 읽기
  - 주제 탐색
  - 용어 탐색
  - 퀴즈
  - 기록
- 읽기 화면은 단순하게 유지

### 6. UX 원칙
- 읽기 흐름 방해 금지
- 정보는 “요청 시 노출”
- 사이드바 상시 노출 금지
- 길이는 문제가 아니라 “통제 가능성”이 중요

### 7. 오디오 기능
- 보조 기능으로 제공
- 작게, 눈에 띄지 않게
- 하단 미니 플레이어 형태
- 자동재생 금지

### 8. 인스타 전략
- 릴스 + 사진 병행
- 릴스 70%, 사진 30%
- 순차 업로드 ❌ → 다양한 방식 반복 ⭕
- 핵심 콘텐츠:
  - 색칠 영상
  - before/after
  - 참여형 질문

### 9. 전환 구조
- 인스타 → 프로필 → 링크 → 체험 → 구매
- 무료 체험 필수
- CTA 문구:
  - “무료 체험”
  - “지금 시작”

### 10. 랜딩페이지 구조
- 첫 화면 = 메시지 + CTA
- 바로 다운로드
- 짧은 사용법
- 실제 사용 이미지
- 다시 CTA

## ❗ 미결사항

### 1. 12색 컬러 시스템
- 정확한 색상 코드 정의 필요
- 인쇄 vs 화면 색 차이 고려 필요

### 2. 의미 단위 분리 기준
- 자동화 가능 여부
- 편집 기준 정리 필요

### 3. 책 디자인 최종 확정
- 여백 mm
- 줄간격
- 폰트 선택

### 4. PDF 제작 방식
- 자동 생성 vs 수작업
- 템플릿 시스템 필요

### 5. 웹 기능 우선순위
- MVP 범위 결정 필요
- 읽기 vs 퀴즈 vs 기록

### 6. 수익 모델
- 책 판매 중심인지
- 구독 모델 추가 여부
